# Setup Wizard Skill

This skill is for **first-time setup**. It is designed to be run entirely inside the IDE chat — no terminal, no API keys, no programming knowledge required. The AI model conducts the interview, then writes the configuration files directly.

---

## When to invoke

Run this skill when:
- A new user has just cloned the project.
- The user says "set me up", "run the wizard", or "I'm new, help me configure this".
- `config_user.py` or `data/user_profile.md` do not exist yet.

---

## Step-by-Step Execution Protocol

### Step 0 — Welcome
Greet the user warmly. Briefly explain that you will ask a few questions and then automatically configure the system. Tell them they do **not** need to touch any code.

---

### Step 1 — Collect Resume

Ask the user to provide their resume in **any** of these formats (your choice):

- **Drag & drop or attach** a PDF or Word file directly into this chat.
- **Paste** the text content of their resume.
- **Give you a file path** if the resume is already saved locally (e.g., `data/base_resume.md`).

If they provide a file path, use your file-reading tools to load it.
If they provide a PDF or doc attachment, read the content visually.
If they paste text, use that directly.

Confirm you have received the resume before proceeding.

---

### Step 2 — Preferences Questionnaire

Ask the following questions **one at a time** (conversationally, not as a form). Wait for the answer to each before asking the next. Keep the tone friendly and non-technical:

1. **Target Roles:** "What job titles are you hoping to land? (e.g., Product Manager, Clinical AI Lead, Operations Director)"
2. **Industry:** "Any specific industries? Healthcare, fintech, SaaS, open to anything?"
3. **Key Skills:** "What are the 3-5 skills you want employers to see first on your profile?"
4. **Minimum Salary:** "What is the minimum monthly salary in USD you would accept? Just give me a number."
5. **Work Style:** "Remote only, hybrid, or open to on-site?"
6. **Seniority:** "What seniority level are you targeting? (Junior, Mid-Level, Senior, Director/VP, open)"
7. **Avoid:** "Are there any role types or keywords you want the system to NEVER show you? (e.g., sales, patient care, on-call)"

---

### Step 3 — Generate User Profile

Using the resume and preferences collected, write a `data/user_profile.md` file in this exact format:

```markdown
Name: [Name from resume, or "User" if not found]

Background:
- [Most relevant experience bullet — translate clinical/operational experience into business value]
- [Second strongest relevant experience]
- [Third strongest relevant experience]
- [Additional context if applicable]

Technical / AI Skills:
- [Top skill]
- [Second skill]
- [Additional skills as bullets]

Career Goals:
- Salary requirements: Minimum $[salary] USD monthly
- Roles targeted: [titles from preferences]
- Industry: [industry from preferences]
- Work style: [remote/hybrid/onsite]
- Seniority: [seniority]
- Avoid: [avoid list]
```

**Writing rules:**
- Be concise and factual — this profile is injected into every AI scoring prompt.
- Translate clinical/medical language into business value where applicable.
- Do NOT invent facts. If unsure, use neutral language.

Use the `Write` tool to save this file to `data/user_profile.md`.

---

### Step 4 — Generate Search Queries

Based on the target roles, skills, industry, work style, and avoid list, generate **10–15 highly specific boolean search queries**.

Rules:
- Include the target title in quotes for exact matching.
- Include `remote` if the user wants remote work.
- Use a minus sign (`-`) to exclude things from the avoid list.
- Keep a mix: some broad, some very niche.
- Only standard boolean: quotes for exact match, minus for exclusion.

---

### Step 5 — Write config_user.py

Use the `Write` tool to create `config_user.py` at the project root with this exact structure:

```python
"""
User-specific Configuration Overrides
Generated by Setup Wizard (IDE Chat)
"""

# ─── Job Search Queries ───────────────────────────────────────────────────────
SEARCH_QUERIES = [
    # [Add each query as a string in this list]
]

# ─── Scoring & Salary Preferences ─────────────────────────────────────────────
SCORE_SHOW_THRESHOLD    = 50  # Jobs >= this appear in dashboard (0-100 scale)
SCORE_ARCHIVE_THRESHOLD = 49  # Jobs <= this are auto-archived (0-100 scale)

MIN_SALARY_USD_MONTHLY = [salary_number]
PENALIZE_MISSING_SALARY = False
```

---

### Step 6 — Confirmation & Next Steps

After writing both files, confirm to the user what was created. Then tell them the **three remaining manual steps**:

1. **Set your API key:**
   - Find the file named `.env.example` in the project.
   - Rename it to `.env`.
   - Get a free Gemini API key at `https://aistudio.google.com/app/apikey`.
   - Paste the key into the `.env` file and save.

2. **Save your full resume:**
   - Save your complete resume as `data/base_resume.md` (Markdown text format).
   - This is used for generating tailored resumes for specific jobs.

3. **Start the job hunt:**
   - In VS Code, go to **Terminal → Run Task...**
   - Select **"Discover & Score Jobs"**.
   - The system will start finding and scoring jobs automatically.

## File Outputs

| File | Purpose |
|---|---|
| `data/user_profile.md` | Profile injected into all AI scoring prompts |
| `config_user.py` | Search queries + salary thresholds (overrides `config.py` defaults) |

Both files are gitignored and stay private on the user's machine.
