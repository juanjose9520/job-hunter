# Job Hunt — AI Assistant Context

This is an automated, AI-powered job hunting system that helps users discover, score, and track remote job opportunities. You are the AI assistant helping to operate and extend this tool.

---

## Project Overview

The system automatically:
1. **Discovers** remote jobs from multiple job boards (APIs + scraping).
2. **Scores** each job using Gemini AI against the user's profile (0–100 scale).
3. **Presents** results in a browser-based dashboard for review.
4. **Tailors** resumes and cover letter seeds for shortlisted jobs.

---

## User Profile

**Always read `data/user_profile.md` to understand who this user is before taking any action.**

This file contains their background, target roles, key skills, salary requirements, and what to avoid. It is gitignored and unique to each installation — never assume you know the user's profile from memory or from the repository defaults.

If `data/user_profile.md` does not exist, the user has not completed setup yet. Follow the **Setup Wizard** skill to onboard them first.

## Key Files

| File | Purpose |
|---|---|
| `run.py` | Main CLI entry point — all commands go through here |
| `config.py` | Central configuration — queries, thresholds, paths, API keys |
| `config_user.py` | User overrides (gitignored) — generated by Setup Wizard |
| `data/user_profile.md` | User profile injected into all AI prompts (gitignored) |
| `data/base_resume.md` | Full base resume used for tailoring (gitignored) |
| `data/.personal_info` | Private user info (gitignored) |
| `data/jobs.db` | SQLite database of all discovered jobs (gitignored) |
| `scripts/score.py` | Gemini-powered job scoring logic |
| `scripts/database.py` | All database read/write operations |
| `dashboard/` | Browser-based job review dashboard |

## Common Commands

```bash
# Run full pipeline: discover + score
python run.py discover

# Deep scrape (slower, more thorough)
python run.py discover --deep

# Re-score all pending jobs
python run.py score

# Tailor resume for a job (requires DB job ID)
python run.py tailor --job <ID>

# Generate LinkedIn rewrite
python run.py linkedin

# Launch dashboard
python run.py dashboard
```

---

## Configuration Override System

`config.py` loads defaults (search queries, thresholds, salary floor). If `config_user.py` exists at the project root, its values **override** the defaults automatically. This is how the Setup Wizard personalizes the system without requiring the user to edit `config.py`.

---

## Privacy & Security

The following are **gitignored** and never leave the user's machine:
- `.env` (API keys)
- `data/*` except example files
- `output/` (generated resumes, cover letters)
- `config_user.py` (personal search config)
